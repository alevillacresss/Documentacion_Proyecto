Documentación del Proyecto

Bienvenido a la documentación de nuestro proyecto. En esta página encontrarás todos los recursos e información necesaria para entender y trabajar con las herramientas y el desarrollo que hemos implementado.

En esta documentación, hemos dividido el contenido en varias secciones clave, que te permitirán comprender el uso de la aplicación, su instalación y los detalles técnicos sobre cómo interactuar con el sistema.

Secciones:

  • Manual de Usuario: Guía básica para que puedas comenzar a utilizar la aplicación.
  • Guía de Referencia: Información detallada sobre las funcionalidades y capacidades del sistema.
  • Guía de Instalación: Pasos para instalar y poner en funcionamiento la aplicación en diferentes plataformas.

Reglas de Uso con MkDocs

MkDocs es una herramienta de documentación estática que permite la creación de sitios web de documentación de manera fácil y rápida. Algunas de las reglas y configuraciones esenciales para trabajar con MkDocs incluyen:

  1. Archivos Markdown: La documentación se estructura mediante archivos .md (Markdown). Este formato es fácil de leer y escribir, ideal para mantener la claridad en los documentos.

  2. Configuración de mkdocs.yml: En el archivo mkdocs.yml, se configuran aspectos como el nombre del sitio, el tema (por ejemplo, "material"), y la estructura de navegación entre los documentos.

  3. Construcción del Sitio: Con el comando mkdocs build se genera el sitio estático y mkdocs serve permite visualizar la documentación de manera local en tu navegador.

  4. Publicación en GitHub Pages: El sitio generado puede ser fácilmente desplegado en GitHub Pages para que tu documentación esté disponible públicamente en línea.
